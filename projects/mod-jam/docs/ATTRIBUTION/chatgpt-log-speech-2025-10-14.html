If you want to process speech with WebAudio effects (grittier / more 90s)

Workflow: generate audio (meSpeak can produce raw wav data or just play), decode it into an AudioBuffer, then route through an AudioContext graph with effects: a bitcrusher (reduce bit depth / sample rate), ring modulator (multiply by a carrier), and optional low-pass/high-pass to shape the timbre.

Example effect nodes (bitcrusher + ring modulator ‚Äî drop into any WebAudio AudioContext):

// create an AudioContext and simple DSP nodes
const ctx = new AudioContext();

function makeBitcrusher(bits = 6, normFreq = 0.25) {
  const node = ctx.createScriptProcessor(4096, 1, 1);
  const step = () => Math.pow(0.5, 16 - bits);
  node.onaudioprocess = function(e) {
    const input = e.inputBuffer.getChannelData(0);
    const output = e.outputBuffer.getChannelData(0);
    const stepVal = step();
    for (let i = 0; i < input.length; i++) {
      // simple sample-and-hold / quantization
      output[i] = Math.round(input[i] / stepVal) * stepVal;
    }
  };
  return node;
}

function makeRingModulator(freq = 30) {
  // ring modulation via GainNode with an oscillator as gain control
  const gain = ctx.createGain();
  const osc = ctx.createOscillator();
  const oscGain = ctx.createGain();
  osc.type = 'sine';
  osc.frequency.value = freq;
  oscGain.gain.value = 0.5; // modulation depth
  osc.connect(oscGain);
  oscGain.connect(gain.gain);
  osc.start();
  return { input: gain, output: gain }; // connect source -> gain -> rest of graph
}

How can I set the default value to Fred?

Good question ‚Äî you can set Fred as the default selection very simply once the voices are loaded. Here‚Äôs how to do it safely (accounting for the fact that voices load asynchronously on many browsers):

<select id="voice"></select>

<script>
function populateVoices() {
  const select = document.getElementById("voice");
  select.innerHTML = "";

  const voices = speechSynthesis.getVoices();
  for (let voice of voices) {
    const opt = document.createElement("option");
    opt.value = voice.name;
    opt.textContent = `${voice.name} (${voice.lang})${voice.default ? " ‚Äî DEFAULT" : ""}`;
    select.appendChild(opt);
  }

  // Try to preselect ‚ÄúFred‚Äù if available
  const fredVoice = voices.find(v => /Fred/i.test(v.name));
  if (fredVoice) {
    select.value = fredVoice.name;
  } else if (voices.length > 0) {
    // fallback: pick system default
    select.value = voices.find(v => v.default)?.name || voices[0].name;
  }
}

// Voices load asynchronously
speechSynthesis.onvoiceschanged = populateVoices;
populateVoices();
</script>

üîç How it works:

  • It waits for speechSynthesis.onvoiceschanged (since getVoices() might return [] initially).

  • It populates the dropdown.

  • It searches for a voice whose name matches /Fred/i.

  • If found, it sets that option as selected.

  • Otherwise, it falls back to the system default voice or the first one available.

Where is the variable speechSynthesis coming from?

Excellent catch ‚Äî the variable speechSynthesis isn‚Äôt declared anywhere in your code because it‚Äôs a built-in global object provided by the browser.

üåê What it is

window.speechSynthesis is part of the Web Speech API, implemented in all modern browsers (Chrome, Edge, Safari, Firefox, etc.). It‚Äôs automatically available on the window object, so you can access it directly as:

speechSynthesis
// or explicitly:
window.speechSynthesis

üî© What it provides

This object gives you the interface to the browser‚Äôs text-to-speech engine:

Method / PropertyDescription
speechSynthesis.speak(utterance)Starts speaking a SpeechSynthesisUtterance
speechSynthesis.cancel()Stops all ongoing speech
speechSynthesis.pause() / resume()Pause/resume
speechSynthesis.getVoices()Returns an array of available voices
speechSynthesis.onvoiceschangedEvent fired when voices become available

üß† Related object: SpeechSynthesisUtterance

You create this per sentence or paragraph you want spoken:

utterance.rate = 0.9;

That‚Äôs all it takes ‚Äî no imports, no extra libraries.
